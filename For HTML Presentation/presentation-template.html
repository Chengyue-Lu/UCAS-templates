Part 1

问题提出页模板

这一页适合用来交代背景、说明为什么原有方法不够，或定义你真正关心的问题。

02 / 05
01

先讲场景或背景

先把听众带进问题，不要一上来就讲方法或公式。

02

再讲真正困难在哪里

说明不是“不会做”，而是“为什么传统做法会遇到结构性困难”。

03

最后落到核心问题

这一块建议作为承接，顺势把听众带到下一页的方法或研究脉络。

适用写法

左侧讲递进，右侧讲解释

这类双栏页很适合做“现象 vs 含义”“已有做法 vs 研究缺口”“问题 vs 影响”的对照。

注意

不要把副标题写成给自己看的备注

副标题应该服务听众理解，例如说明这页的作用、结论或在整场报告中的位置。

Part 2

图示页模板

这里不用真实图片，只保留占位框，方便你后续替换成论文图、实验图、流程图或截图。

03 / 05
IMAGE / CHART PLACEHOLDER
把这里替换成论文图、方法图、结果图或重绘图
Point 1

先讲这张图在说明什么

不要先逐细节解释。应该先告诉听众，这张图在整场报告里承担什么作用。

Point 2

再讲最关键的结构或差异

只挑支撑主线的部分讲，不要把复杂图完整照念一遍。

Point 3

最后落到你的结论

用一句话讲清：这张图为什么能支持你的问题、方法或实验解释。

模板提示：复杂原图不一定要整张搬上来，很多时候裁图或重绘更适合课堂汇报。
Part 3

并列对照页模板

适合用来展示研究脉络、方法模块、实验平行组，或者多个对照维度。

04 / 05
Block A

问题背景

这里可以写任务背景、研究起点或第一个关键节点。

Block B

已有推进

这里适合放相关工作、前作方法或上一阶段结论。

Block C

研究缺口

这里写为什么前面的推进还不够，以及还缺哪一步。

Block D

你的切入点

最后一块通常用来落到本文、你的方案，或者你想强调的创新。

总结页模板

总结页建议提升问题层次，不建议再引入新的核心内容。

传统视角

最后记忆点
