带目录栏的 HTML 展示模板

这份版本适合“章节式”展示。左侧常驻目录，右侧内容区按页浏览；每一页内部保留滚动，适合长内容、图文混排和逐段讲解。

适合的场景

  • 课程汇报、调研报告、项目展示
  • 需要目录感和章节切换的长内容
  • 每页内容较多，希望页内继续滚动阅读

快速特性

  • 左侧目录点击切页
  • 每页独立滚动
  • 无外部依赖，自包含单文件
公式支持 代码高亮 主题切换
提示：如果你之后想做得更像“网页课件”，这一版比传统 Beamer 更容易扩展动画、图表、交互和图像布局。
Chapter 01

为什么把展示改成 HTML

LaTeX Beamer 的优势是排版稳、公式强、学术感足，但它在复杂图形、网页交互和灵活布局方面会比较受限。HTML 更像一个“可编程的展示容器”，适合你后面不断往里加图表、动效、交互按钮和嵌入式媒体。

Beamer 更擅长
  • 数学公式和学术排版
  • 统一严谨的版式
  • PDF 输出与打印
HTML 更擅长
  • 灵活布局和复杂图形
  • 局部滚动与目录导航
  • 动画、交互和多媒体嵌入
如果你的展示内容需要“像网页一样”承载很多模块，HTML 基本会比 Beamer 更顺手。
Chapter 01.2

目录栏的价值

定位清晰

用户始终知道自己在整份展示中的位置，适合章节式汇报。

跳转快速

不需要连续翻页查找，点击目录就能直接到目标部分。

结构可见

比纯幻灯片更像“内容索引”，特别适合长报告。

Chapter 02

页面内滚动的阅读体验

这一页故意放得长一点，模拟“一个页面里有很多小节”的情况。你滚轮时，优先在本页内部上下移动；只有到达顶部或底部时，才会自动切换上一页或下一页。

适合放什么
  • 长段解释文字
  • 多张图和注释
  • 表格、代码、流程图
交互方式
  • 滚轮滚动内容
  • 目录栏点击切换页面
  • 键盘方向键辅助翻页
如果你希望目录栏固定、内容区按页面滚动，这个结构比 Beamer 更接近“文稿 + 导航”的组合。
Chapter 02.1

长内容示例

第一段：可以在这里放更细的说明，展示问题背景、方法步骤或实验设置。由于内容区可滚动，所以不必强行把所有信息压缩在一屏。

第二段：当页面里存在图表、说明文字、对比段落和结论时，HTML 允许你按照阅读逻辑组织，而不是受限于固定幻灯片框架。

第三段：如果后续你想做目录联动、高亮当前章节、自动生成章节列表，都可以继续往这个结构上加，不需要推翻重来。

第四段：你也可以在这里放多图排版，比如左边图、右边说明，或者多列卡片。HTML 的 flex 和 grid 都很适合这种东西。

第五段：滚动继续往下，可以显示更长的内容。这个模板的目标就是让“信息密度大但不拥挤”成为可能。

第六段：如果你希望这一页更像论文报告，也可以把这一页拆成多段小标题，目录栏依然能维持整体结构感。

第七段：长内容在演示时并不一定是坏事。只要导航清晰、视觉层次明确，它会比过度压缩的幻灯片更容易讲清楚逻辑。

第八段：继续滚动可以体验到内容区的独立滚动效果。到达底部后，再往下滚会自动切到下一页。

图形

结论与下一步

如果你的重点是“展示逻辑清晰、目录感强、内容不局限在一屏”，那这一版结构会比传统幻灯片更适合。它既保留了演示的节奏，也保留了网页的自由度。
